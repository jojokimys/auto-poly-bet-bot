# Polymarket 유동성 리워드 프로그램 리서치

> 조사일: 2026-03-15

---

## 1. 유동성 리워드 프로그램 개요

Polymarket은 마켓에 유동성을 제공하는 사람(Maker)에게 매일 USDC 리워드를 지급한다. 핵심 원리는 **미드포인트(중간가격)에 가까운 양방향 지정가 주문을 올려놓으면 리워드를 받는 것**이다.

### 리워드 지급 구조
- **지급 시점**: 매일 UTC 자정 (한국시간 오전 9시)
- **최소 지급액**: $1 이상이어야 지급
- **샘플링**: 매분 스냅샷 촬영, 에폭당 10,080개 샘플
- **보상 풀**: 마켓별로 설정된 일일 리워드 풀을 모든 LP가 Q스코어 비율로 나눠 가짐

### 보상 종류 (3가지)
1. **Polymarket 기본 유동성 리워드** - 플랫폼이 지정 마켓에 할당
2. **스폰서 리워드** - 누구든 특정 마켓에 추가 리워드 예치 가능 (중첩 가능)
3. **홀딩 리워드** - 포지션 보유 시 연 4% 이자 (별도)

## 2. Q스코어 공식 (핵심)

### 기본 스코어 함수
```
S(v, s) = ((v - s) / v)² × b
```
- `v` = 마켓별 최대 스프레드 (예: 3센트)
- `s` = 조정된 미드포인트로부터의 실제 거리
- `b` = 주문 사이즈 (쉐어 수)

### 양방향 스코어 집계
- **Q_one**: 프라이머리 마켓 bid + 보완 마켓 ask
- **Q_two**: 프라이머리 마켓 ask + 보완 마켓 bid
- **Q_min = min(Q_one, Q_two)** → 양쪽 중 작은 쪽이 최종 점수

### 단방향 패널티
- **미드포인트 0.10~0.90**: 단방향도 가능하지만 점수 1/3로 축소
  - `Q_min = max(min(Q_one, Q_two), max(Q_one/3, Q_two/3))`
- **미드포인트 <0.10 또는 >0.90**: 반드시 양방향 필요

### 핵심 인사이트: 2차함수(Quadratic)의 의미
**경쟁자보다 2배 타이트하게 호가를 놓으면, 리워드는 ~4배가 된다.** 이것이 전문 마켓메이커가 계속 스프레드를 좁히는 인센티브다.

## 3. 실전 수익 기대치

### 자본 규모별 예상 수익

| 자본 | 마켓 수 | 일일 스프레드 수익 | 월 리워드 | 월 총수익 | 연 ROI |
|------|---------|-------------------|----------|----------|--------|
| $10K | 2개 | $40-80 | $200-500 | $1,400-2,900 | 14-29% |
| $50K | 6-8개 | $180-350 | $800-2,000 | $6,200-12,500 | 15-30% |
| $200K | 15-25개 | $600-1,200 | $3,000-8,000 | $21,000-44,000 | 12-26% |

### 구체적 사례
- **4일간 $8,000**: 스폰서 LP 리워드 ~$6,000 + 트레이딩 수익 ~$2,000
- **전문 마켓메이커**: 일일 거래량 $100K+ 마켓에서 하루 $150-300/마켓
- **$10K 자본 현재 환경**: 하루 $30-80 (2024-25년 피크 $200-300에서 하락)
- **잘 관리된 시스템**: 총 스프레드 수익의 60-75%가 순이익 (인벤토리 리스크 차감 후)

### 수익 경쟁 현황
- $100+ 누적 수익 = 전체 지갑 상위 6%
- $650+ 누적 수익 = 전체 지갑 상위 2% (~1,000개 지갑)
- Polymarket 전체 트레이더 중 ~16%만 흑자 (84% 적자)

## 4. 마켓 선택 전략 (수익의 50% 이상 결정)

### 좋은 마켓 조건
- **일일 거래량 $50K+**: 충분한 유동성
- **30일+ 장기 마켓**: 2028 대선 등 오래 지속되는 이벤트
- **안정적 확률 변동**: YES 확률이 급변하지 않는 마켓
- **높은 인센티브 대비 유동성 비율**: 절대 리워드 금액이 아니라 "유동성 1달러당 리워드"
- **명확한 해결 기준**: 애매한 해결 조건은 리스크
- **"조용한 기간"**: 당분간 큰 뉴스가 없는 시기

### 피해야 할 마켓
- 해결 임박 마켓 (48시간 이내)
- 극단적 확률 ($0.05 또는 $0.95 근처)
- 이벤트 직전 마켓 (발표/결과 24시간 전)
- 유동성이 너무 얇은 마켓 ($1,000 주문이 10포인트 움직이는 수준)

### 실전 팁
- 기존 유동성이 수천 쉐어 수준인 마켓 선호 → 내 주문이 다른 사람과 함께 "풀링"되어 체결 확률 낮아짐
- 오래 체결 안 되면서 리워드만 쌓이는 게 이상적

## 5. 호가 배치 전략 (Spread Placement)

### 계층적 주문 구조 (Tiered/Laddered)
```
미드포인트 근처: 소량 주문 (좁은 스프레드) → 고빈도 체결, 높은 Q스코어
약간 먼 곳: 중간 주문 → 버퍼 역할
먼 곳: 대량 주문 → 비상 방어선
```

### 밸런스 상태별 스프레드 조정

**균형 상태 (인벤토리 중립)**:
- Bid: $0.48 / Ask: $0.52
- 스프레드: $0.04 (대칭)

**YES 초과 보유 시 (스큐잉)**:
- Bid: $0.49 (올려서 매도 유도) / Ask: $0.54 (올려서 매수 억제)
- 스프레드: $0.05 (비대칭)

### 동적 스프레드 조정 규칙
- **변동성 급등 시**: 스프레드 즉시 확대 또는 주문 취소
- **주요 발표 직후**: 처음 15분간 스프레드 확대
- **해결 48시간 전**: 노출 축소
- **안정기**: 스프레드 축소하여 Q스코어 극대화

## 6. 인벤토리 리스크 관리 (가장 중요한 리스크)

### 왜 위험한가
Polymarket 컨트랙트는 0 또는 1로 결제된다. YES를 잔뜩 들고 있는데 NO로 결제되면 **전액 손실**. 이것이 주식 마켓메이킹과 가장 큰 차이.

### 포지션 한도 체계
| 인벤토리 범위 | 상태 | 대응 |
|--------------|------|------|
| ±$500 | 정상 | 소폭 호가 조정 |
| ±$500~$1,500 | 경고 | 대폭 스큐잉 |
| ±$1,500+ | 위험 | 적극적 포지션 축소 또는 일시 중단 |

### 인벤토리 관리 기법
1. **호가 스큐잉**: 과다 보유 방향의 호가를 불리하게 조정
2. **포지션 캡**: 자본의 30-40% 초과 시 호가 완전 철회
3. **시간 기반 축소**: 해결일 가까워질수록 스프레드 확대
4. **멀티마켓 분산**: 단일 마켓에 전체 자본의 15% 이상 금지
5. **현금 보유**: 40% 현금을 인벤토리 교정용으로 보유

### 인벤토리 정리 방법
1. **배치 매도**: 한쪽 포지션을 분할 매도
2. **Merge**: 양쪽 포지션을 합쳐서 USDC로 전환 (YES + NO = $1)
3. **Split**: USDC를 YES/NO 쌍으로 분리하여 부족한 쪽 보충

### 목표
- 일말 기준 **90%+ 인벤토리 중립** 달성
- "포지션 불균형이 대부분의 마켓메이커를 무너뜨리는 진짜 이유"

## 7. Adverse Selection (역선택) 대응

### 역선택이란
정보를 가진 트레이더가 가격 변동 직전에 내 호가를 체결시키는 것. 내가 모르는 뉴스를 아는 사람이 내 호가를 때리고, 가격이 불리한 방향으로 움직이면 손실.

### 역선택 징후
- 한 방향 주문 흐름이 지속
- 대형 마켓 오더가 전체 호가 사이즈를 쓸어감
- 체결 후 가격이 계속 불리한 방향으로 이동
- 설명 불가한 거래량 급증

### 방어 전략
1. **주요 이벤트 24시간 전 운영 중단**
2. **뉴스 직후 15분간 스프레드 대폭 확대**
3. **해결 48시간 전 노출 축소**
4. **급격한 가격 변동 시 즉시 호가 취소** → 회복을 기대하지 말 것
5. **오더북 구조가 안정될 때까지 재진입 대기**
6. **뉴스 모니터링 + 자동 서킷브레이커** 설정

## 8. Q스코어 최적화 / 게이밍 전략

### 기본 원칙
1. **양방향 필수**: 단방향은 3배 패널티 → 항상 bid+ask 동시 유지
2. **최대한 타이트하게**: 2차함수이므로 1센트 더 가까우면 기하급수적 보상 증가
3. **연속 유지**: 갭이 생기면 스코어링 리셋 → 자동화 시스템 유리
4. **사이즈 비례**: 주문 크기에 비례하여 스코어 증가 (선형)

### 고급 최적화
- **조정된 미드포인트(Adjusted Midpoint) 추적**: 실제 미드포인트가 아니라 Polymarket이 계산한 조정 미드포인트 기준
- **경쟁자 분석**: 같은 마켓의 다른 LP 호가를 모니터링하여 더 타이트하게 배치
- **마켓별 max spread 확인**: API로 마켓별 최대 스프레드와 최소 사이즈 컷오프 확인
- **체결 알림 활용**: 체결되면 즉시 반대쪽에 LP하여 추가 볼륨/수수료 캡처

### 시간 활용
- 적극 운영: 10-20% (배치, 조정, 교정)
- 패시브 운영: 80-90% (자동 실행)
- 단, 시장 변화에 즉각 대응할 준비는 항상 필요

## 9. 리워드 파밍 vs 체결 회피의 트레이드오프

### 딜레마
- **타이트한 호가** → Q스코어 높음 → 리워드 많음 → 하지만 체결 확률도 높음 → 인벤토리 리스크
- **넓은 호가** → Q스코어 낮음 → 리워드 적음 → 체결 확률 낮음 → 안전

### 실전 해법
1. **조용한 마켓 선택**: 가격 변동이 적으면 타이트해도 안전
2. **깊은 유동성 마켓**: 다른 LP와 함께 풀링되어 내 주문만 체결될 확률 낮음
3. **소량 타이트 + 대량 와이드**: 미드 근처에 소량으로 Q스코어 확보, 먼 곳에 대량으로 방어
4. **이벤트 기반 동적 조정**: 평소 타이트 → 이벤트 시 와이드 또는 일시 중단

### 핵심 결론
> "강력한 독자적 알파가 없다면, 리워드를 메인 수익원이 아니라 보너스로 취급하는 것이 건강하다."

## 10. 필요 인프라 & 도구

### 기술 스택
- **Polymarket CLOB API**: REST + WebSocket (50ms 이하 레이턴시)
- **Python 클라이언트**: `py-clob-client` (공식)
- **API 제한**: 분당 약 60개 주문
- **가스비**: Polygon 기반, 트랜잭션당 $0.001-0.01

### 유용한 외부 도구
- **Betmoar**: 마켓 스크리닝, 리워드 밀도 분석
- **Polycule (TG Bot)**: 지갑 추적, 체결 알림
- **PolyRewards**: 리워드 수익 추적
- **PolyAlertHub**: 체결 알림 → 즉시 반대 포지션 진입용
- **opt.markets**: 리워드 최적화 도구

### 확장 단계
1. **학습기** ($10K): 1-2개 마켓, 수동 관리
2. **확장기** ($50K): 5-8개 마켓, 스프레드시트 도구
3. **전문기** ($200K+): 15개+ 마켓, 전면 API 자동화

## 11. 실전 전략 요약: 우리 봇에 적용할 포인트

### 즉시 적용 가능한 전략
1. **리워드 마켓 스캐닝**: 리워드 풀 대비 경쟁 유동성이 낮은 마켓 자동 탐색
2. **양방향 지정가 주문**: Q스코어 극대화를 위한 bid+ask 동시 배치
3. **계층적 주문**: 미드 근처 소량 + 먼 곳 대량
4. **인벤토리 모니터링**: 한쪽 포지션 과다 시 자동 스큐잉
5. **이벤트 감지**: 주요 뉴스/이벤트 전후 자동 스프레드 조정

### 리스크 관리 파라미터
- 단일 마켓 최대 자본: 전체의 15%
- 현금 보유: 40%
- 인벤토리 캡: 자본의 30-40%에서 호가 철회
- 해결 48시간 전 노출 축소
- 이벤트 24시간 전 운영 중단

### 수익 목표 (보수적)
- $10K 자본 기준: 월 $1,000-2,000 (10-20% 월수익)
- 스프레드 수익 + 리워드 합산
- 60일 이상 일관된 양수 수익 확인 후에만 스케일업
